10 Mobile UX design principles

  • Content Prioritization
    • Grab users attention fast
    • Less is more, and allow the user to engage with ease
    • Display only essential content, especially on smaller screens
    • Any additional content should be available through a menu
    • Menus should be simple, and reduced to icons where it makes. sense
  • Making Navigation Intuitive
    • If something is complex, the user will likely go to a different site, or product
    • Have all the information the user needs readily available
    • Make it known where they are in your site without having to guess
  • Touchscreen target sizes
    • give enough space and size for the user to be able to interact with an icon or button that there isnt a chance they will click on another link.
    • Include an undo button in the site incase of possible miss clicks

  • Provide User Control
    • Allow multiple layouts for the user to control how they want it to look.
    • Allow them to control notifications
    • Give the user any option that might be helpful, and let them choose for themselves
  • Legible Text Content
    • If users cant easily read your content, there is no point to your content
    • Anything smaller than 11 points becomes challenging for any screen
    • Large fonts just become annoying to read
    • A good rule of thumb is to use 30–40 characters per line for mobile.
    • Space different sections of text out so the user can take everything in
  • Make Interface Elements Clearly Visible
    • Have sufficient contrast between content and background in your designs so it’s legible, in any setting, even outside in the sunlight
    • WC3’s Web Content Accessibility Guidelines -
  • Hand Position Controls
    • 49% of people rely on 1 thumb while using their mobile device
    • Common features should be placed in easy to reach areas
    • Buttons like Delete should be put in hard to reach areas to avoid error
    • Add accessibility behaviors to allow the user to decide how they want to navigate the interface.
    • Keep in mind, left-handed users as well
  • Minimize Data Input
    • Shorten forms
    • using ‘remember me’ option to avoid multiple login’s
    • Provide Autocomplete where it makes sense
    • Search history and location detection
    • Change keyboard layouts where possible
      • A # layout for phone number or account number entry
    • Drop down select where possible
  • Create a seamless experience
    • Make sure to use mobile shortcuts to speed things up
      • ocr for cards
      • gps for locations
      • Biometric logins
    • Change your layout based on screen size but provide the same experience across.
    • Sync changes between the different experiences allowing users work not to be interrupted if they decide to switch mediums.
  • Test your design
    • Continually optimize your site
    • Test on multiple devices and screens
      • Nothing is worse than a slow load time on one device but smooth on another
    • Talk to users and get feedback for what to change
